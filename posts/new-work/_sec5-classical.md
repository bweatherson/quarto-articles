## Is This Still Classical Logic? {#sec-classical}

One might worry that by adopting possibility semantics, I'm abandoning classical logic. After all, I've introduced incomplete possibilities where some propositions are neither true nor false. Doesn't that already put us in the realm of many-valued logic? I'll argue that the answer depends on what we mean by "classical logic," and that on most reasonable interpretations, possibility semantics preserves it.

The trouble is that "classical logic" is ambiguous. Let me distinguish several options, each representing a different way of characterizing a logical system.

**Option 1: Classical logic as a set of theorems.** We can identify a logic with the set of formulas that are theorems—formulas that are true in all models. Possibility semantics clearly preserves this. Every tautology of classical logic remains true under the Humberstone semantics (where a formula is true at a possibility iff it's true in at least one of its refinements). Since classical tautologies are true in all standard classical models, and every standard classical model is a possibility semantics model (one where each possibility is a complete classical world), all classical theorems are preserved.

**Option 2: Classical logic as axioms plus rules.** We can identify a logic with a set of axioms and derivation rules. The standard axiomatization of classical propositional logic includes modus ponens: if $A$ and $A \to B$ are theorems, then $B$ is a theorem. Possibility semantics preserves modus ponens. And of course it preserves the axioms (they're among the theorems). So on this characterization too, classical logic is preserved.

**Option 3: Classical logic as single-conclusion sequents.** We can take a logic to be a set of valid single-conclusion sequents: statements of the form $\Gamma \vdash A$, meaning that whenever all members of $\Gamma$ are true, $A$ is true. As I showed in @sec-humberstone, possibility semantics validates exactly the single-conclusion sequents of classical logic. This should be our baseline.

**Option 4: Classical logic as basic sequents plus derivation rules.** Here we identify a logic with a set of basic sequents—at minimum, $\Gamma \vdash A$ whenever $A \in \Gamma$—plus rules for generating further valid sequents. The most important rules include modus ponens, but also conditional proof: if $\Gamma, A \vdash B$, then $\Gamma \vdash A \to B$. And cut: if $\Gamma \vdash A$ and $\Gamma, A \vdash B$, then $\Gamma \vdash B$.

Possibility semantics preserves all of this. We have the basic sequents, and all the standard derivation rules are sound. This is where an important difference emerges with supervaluationism. Williamson [-@Williamson1994] showed that supervaluationism fails to validate conditional proof. That is, there are cases where $\Gamma, A \vdash B$ is valid under supervaluationism but $\Gamma \vdash A \to B$ is not. This is a significant limitation. Possibility semantics avoids it.

**Option 5: Classical logic as multiple-conclusion sequents.** We can also identify a logic with the set of valid multiple-conclusion sequents: statements $\Gamma \vdash \Delta$ (where $\Delta$ is now a set, not a single formula), meaning that whenever all members of $\Gamma$ are true, at least one member of $\Delta$ is true. Here possibility semantics and classical logic diverge.

In classical multiple-conclusion logic, the sequent $\vdash p, \neg p$ is valid—this is the multiple-conclusion form of the law of excluded middle. It says: with no premises, either $p$ or $\neg p$ must be true. Under possibility semantics, this fails. There exist possibilities where $p$ is neither true nor false (neither $p$ nor $\neg p$ holds in any refinement). So the sequent $\vdash p, \neg p$ is not valid in possibility semantics.

More generally, possibility semantics does not validate classical multiple-conclusion logic.

**Option 6: Classical logic as basic multiple-conclusion sequents plus rules.** Just as we distinguished basic single-conclusion sequents from those generated by rules, we can distinguish basic multiple-conclusion sequents from those derived via rules. Possibility semantics fails to validate these as well.

Let me now present a comparison. The following table shows which of these six characterizations are preserved by possibility semantics and by supervaluationism:

| Characterization | Possibility Semantics | Supervaluationism |
|---|---|---|
| Set of theorems | ✓ Yes | ✓ Yes |
| Axioms + modus ponens | ✓ Yes | ✓ Yes |
| Single-conclusion sequents | ✓ Yes | ✓ Yes |
| Basic sequents + all rules (incl. conditional proof) | ✓ Yes | ✗ No |
| Multiple-conclusion sequents | ✗ No | ✗ No |
| Basic multi-conclusion sequents + rules | ✗ No | ✗ No |

The key difference is Option 4. Possibility semantics preserves the full inferential power of classical single-conclusion logic, including conditional proof. Supervaluationism, by contrast, loses conditional proof. This makes possibility semantics, in a clear sense, *more* classical than supervaluationism.

Now, what should we say about multiple-conclusion logic? This is where the story becomes more complicated.

On one hand, multiple-conclusion logic is philosophically controversial. Dummett [-@Dummett] has expressed skepticism about multiple-conclusion sequents as a way of characterizing logical validity. Wright [-@Wright] and others have noted that the multiple-conclusion framework seems unmotivated from the perspective of intuitionistic logic and other non-classical systems. There's something odd about the idea that a valid argument can have no premises. And rejecting multiple-conclusion logic is not a radical move; it aligns with longstanding philosophical doubts about whether such a framework is the right way to think about logical consequence.

On the other hand, multiple-conclusion logic is deeply embedded in much contemporary work on accuracy and belief. In particular, Joyce's [-@Joyce1998] accuracy dominance arguments—which I'm partly responding to here—typically rely on a claim that can be expressed as a multiple-conclusion sequent. The basic idea is this: one of the complete probability distributions must be true. From no premises, we can infer that the truth is in the set of all complete probability distributions. This is valid in classical logic precisely because it's a multiple-conclusion sequent. In possibility semantics, this inference fails. We reject the claim that exactly one complete state is actual.

So in a sense, my response to Joyce's accuracy dominance argument—via Dempster-Shafer belief functions and incomplete possibilities—involves a non-classical move: the rejection of multiple-conclusion logic. But I want to suggest that this is a species of non-classicality we have reason to accept.

The thought is this. Classical logic, properly understood, should be characterized by Options 1–4 above. These capture the core inferential practices that have made classical logic so powerful and successful. The laws of excluded middle (in its single-conclusion form $\vdash p \vee \neg p$), non-contradiction, and all the standard derivation rules remain valid. We keep modus ponens, conditional proof, cut, and everything else that makes classical reasoning work.

What we lose is the principle that exactly one "complete possible world" (in the classical sense) must be actual. This is not obviously a loss. In fact, it seems like a gain: we avoid the assumption that the world must be fully determinate in every detail, an assumption that has been questioned in metaphysics and epistemology since Peirce. And rejecting it allows us to give a cleaner response to accuracy dominance arguments, one that doesn't require the apparatus of supervaluationism and its attendant costs.

So my answer to the question "Is this still classical logic?" is: yes, it preserves classical logic in all the ways that matter. The one respect in which it deviates—the rejection of multiple-conclusion sequents—represents a defensible restriction of the classical framework, not a wholesale abandonment of it.
